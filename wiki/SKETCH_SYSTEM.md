# Sketch System

## SketchEditorState Resource

`apps/desktop/src/sketch_editor.rs:61` — A Bevy `Resource` that holds all sketch editing state. It exists as a singleton alongside `AppState` and is initialized at startup (`main.rs:40`).

Key fields:
- **active**: Whether the sketch editor is currently open. When `true`, main UI panels are hidden and sketch-specific input and rendering systems run.
- **node_id**: The `GeometryNode` ID of the sketch being edited.
- **document**: A `SketchDocument` — the in-memory 2D sketch data.
- **tool**: The active `SketchTool` (Select, Line, Circle, Arc, Measure).
- **selected_entity / selected_entities**: Entity ID(s) for single and multi-select.
- **plane**: The `SketchPlane` (XY, XZ, YZ) from the sketch definition.
- **line_start / circle_center / arc_center**: Intermediate state for multi-click tools.
- **drag_from / drag_offset**: For point dragging during Select tool.
- **measure_click_a / measure_click_b**: Two-click measurement points.
- **editing_constraint_idx / editing_new_value**: Inline constraint value editing.
- **extrude_distance / extrude_taper_angle**: Extrude parameters shown in the toolbar.
- **undo_stack / redo_stack**: Independent sketch undo/redo (snapshot-based, not Command trait).

## SketchDocument

`kpe_geometry::sketch::document::SketchDocument` — The 2D geometry container holding:

- **lines**: `Vec<Line>` — each line has an id, start point ID, end point ID.
- **circles**: `Vec<Circle>` — center point ID and radius.
- **arcs**: `Vec<Arc>` — center point ID, radius, start angle, sweep angle.
- **points**: `Vec<Point>` — 2D coordinates (x, y) with id.
- **constraints**: `Vec<Constraint>` — geometric constraints applied to entities.

Entity IDs (`EntityId`) are integer handles generated by the document.

## SketchTool Enum

`sketch_editor.rs:89` — Five tools:

| Tool | Behavior |
|---|---|
| `Select` | Pick entities by proximity. Drag points to move them. Click constraint markers to edit values. |
| `Line` | Click for start point, click again for end point. Constraint inference is run automatically on created lines. |
| `Circle` | Click for center, click again for radius. Minimum radius check (0.01). |
| `Arc` | Click for center, click again for sweep end point. Start angle is 0. |
| `Measure` | Click point A, click point B. Displays distance and angle. |

## Coordinate System — 2D to 3D Projection

`sketch_editor.rs:10-32` — Sketches are defined in 2D but drawn in 3D space via a `SketchPlane`:

```rust
pub enum SketchPlane { XY, XZ, YZ }
```

Conversion functions:
- `to_3d(x, y, plane)`: Maps 2D → 3D. XY→(x,y,0), XZ→(x,0,z), YZ→(0,y,z).
- `to_2d(pos, plane)`: Maps 3D → 2D (inverse).
- `sketch_plane_normal(plane)`: Returns the normal direction.
- `circle_basis(normal)`: Computes two orthogonal basis vectors for drawing circles/arcs in 3D.

Ray-plane intersection (`sketch_editor.rs:475`) converts the mouse cursor to a 3D point on the sketch plane. The 3D point is then converted to 2D sketch coordinates.

## Constraint System

`kpe_geometry::sketch::constraints::Constraint` — Geometric constraints between entities:

| Constraint | Meaning |
|---|---|
| `Distance { point_a, point_b, distance }` | Fixed distance between two points. |
| `Angle { line_a, line_b, angle }` | Fixed angle between two lines. |
| `Radius { circle, radius }` | Fixed radius on a circle entity. |
| `Horizontal { line }` | Line must be horizontal (y1 == y2). |
| `Vertical { line }` | Line must be vertical (x1 == x2). |
| `EqualLength { line_a, line_b }` | Two lines must have equal length. |
| `Parallel { line_a, line_b }` | Two lines must be parallel. |
| `Perpendicular { line_a, line_b }` | Two lines must be perpendicular. |
| `Collinear { line_a, line_b }` | Two lines must lie on the same line. |
| `Coincident { point_a, point_b }` | Two points must be at the same location. |
| `Fix { point }` | Pin a point to its current position. |

Constraint markers are rendered as orange spheres in the 3D viewport (via `constraint_pos()` in `sketch_render.rs:191`).

### Solver

The constraint solver uses a gradient descent approach (implemented in `kpe-geometry`). It iteratively adjusts point positions to minimize the total constraint error. The solver is called after every entity creation, deletion, or drag via `editor.document.solve()` (`sketch_editor.rs:557`).

Constraint inference (`editor.document.infer_constraints()`) automatically detects horizontal/vertical/equal-length/perpendicular/coincident relationships when a new line is drawn, based on the geometry relative to existing entities.

### Constraint Value Editing

Users click a constraint marker to open an inline editor window (`sketch_editor.rs:413-455`). Editable constraints are Distance, Angle, and Radius. Changes update the constraint value and re-solve.

## Entity Selection and Dragging

`sketch_editor.rs:544-683` — In Select mode:

1. **Point pick**: The cursor position is tested against all point positions. If within 0.2 units, that point becomes the drag target.
2. **Line pick**: The cursor is tested against all line segments using closest-point-on-segment. If within 0.15 units, the line is selected.
3. **Circle pick**: Distance from cursor to circle perimeter is tested.
4. **Drag**: On mouse-down over a point, a snapshot is saved. On mouse-move, the point position is updated and the solver is re-run. On mouse-release, drag ends.

## Extrude from Sketch Workflow

The sketch toolbar includes an "Extrude" button (`sketch_editor.rs:314`):

1. User configures distance and optional taper angle.
2. On "Extrude", an `ExtrudeDef` node is created referencing the sketch ID.
3. The extrude node is added as a sibling of the sketch node under the same parent.
4. `evaluate_all()` rebuilds the full scene, producing a 3D mesh from the 2D sketch.
5. Selection is moved to the new extrude node.

## Enter/Exit Sketch Mode

- **Enter**: Triggered by double-clicking a Sketch node in the scene tree. This sets `AppState.pending_sketch_edit`, which is picked up by `check_enter_sketch_mode()` (`sketch_editor.rs:687`). The `enter()` method loads existing primitives (Rectangle, Circle, Polygon, Arc) into the `SketchDocument`.
- **Exit**: "Finish" calls `exit()` which converts the sketch document back to `SketchDef` primitives and writes them back to the scene tree via `update_sketch_node()`. "Cancel" simply clears the document and sets `active = false`.

### Sketch Undo/Redo

The sketch editor maintains its own snapshot-based undo (`sketch_editor.rs:130-158`). `save_snapshot()` pushes a clone of the current `SketchDocument` onto `undo_stack` and clears `redo_stack`. Undo pops from `undo_stack` and pushes the current state onto `redo_stack`. Max 100 snapshots.

These sketch edits are **not** integrated with the main document's `CommandHistory`. They affect only the in-memory `SketchDocument` until the user clicks Finish, at which point the sketch definition is written back to the scene tree node as a single atomic update.
